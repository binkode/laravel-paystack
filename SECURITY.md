# Security Policy

We take security seriously at Binkode. If you discover any security-related issues or vulnerabilities in `laravel-paystack`, please follow this policy to report them.

## Supported Versions

Only the latest major/minor release receives active security updates and bug fixes. We recommend keeping your installation up-to-date.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

> [!IMPORTANT]
> Do NOT report security vulnerabilities via public GitHub issues.

If you believe you have found a security vulnerability in this package, please report it privately:

1. **Email:** Send a detailed report to **[myckhel1@hotmail.com](mailto:myckhel1@hotmail.com)**.
2. **Details:** Include the following information in your email:
   - A description of the vulnerability.
   - The version(s) of the package affected.
   - A proof of concept (PoC) or step-by-step instructions to reproduce the issue.
   - Any potential mitigation steps or patches if you have drafted them.

### What to Expect
- **Acknowledgment:** We will acknowledge receipt of your vulnerability report within 48 hours.
- **Resolution:** We will work to resolve the issue as quickly as possible and keep you updated on our progress.
- **Disclosure:** Once a fix is released, we will coordinate public disclosure. We appreciate your cooperation in practicing responsible disclosure.
